Getting Started
What to do on a fresh install before any data has been scraped
  1. Start the server
    Run python main.py and open http://localhost:8000.
  2. Run your first rate scrape
    Go to Rate Intelligence → Now. Click Scrape Now to pull live prices from all 8 competitors (~30–60 s). The dashboard shows mock data until real data exists.
  3. Populate the Forward Rates horizon
    Go to Rate Intelligence → Forward and click Scrape Horizon. Scrapes all 8 competitors × 26 weekly windows (6 months). The daily scheduler keeps it fresh from 07:15 onwards.
  4. Populate Seasonal Analysis
    Go to Rate Intelligence → Seasonal and click Scrape All. Scrapes the 15th of each of the next 12 months (7-night stays). Runs automatically every Monday 08:00 after that.
  5. Check fleet availability across all competitors
    Go to Rate Intelligence → Competitive → Fleet Pressure and click Poll Now. This queries Blue, Lotus, Lava and Go Car Rental for real-time fleet availability across 1-week, 2-week, and 4-week forward windows — showing which specific models are sold out as red pills. Then click Sweep 12 Months to run the full seasonal intelligence sweep: a 12-month availability calendar for Caren + Go Car, plus absence-based inference for Hertz, Avis, and Holdur. Runs automatically twice daily (09:00 and 21:00) with a Monday calendar sweep at 08:30.
  6. Set up SEO tracking (optional)
    Go to Settings → API Configuration, enter your SerpAPI key, and click Save Settings. Then go to SEO Rank Tracker → Check Rankings Now. 10 Iceland car rental keywords are pre-seeded.
  7. Configure price alerts (optional)
    Go to Settings → Price Alerts, paste your Slack webhook URL, set the alert threshold (default 10%), and click Save Alerts Config. Use Test Webhook to confirm delivery.
Automated Schedule
What runs automatically once the server is running
Job When What it does
Rate Scrape Daily 07:00 Live rates for today+7 → today+10, all 8 competitors
Horizon Scrape Daily 07:15 Next 26 weekly windows × 8 competitors; feeds Forward Rates & Booking Window
SEO Check Daily 07:30 Google rankings for all stored keywords via SerpAPI
Alert Check Daily 07:45 Fires Slack alert if any competitor undercuts Blue by the threshold %
Seasonal Scrape Every Monday 08:00 Re-scrapes all 12 monthly anchor dates (15th of each month, 7-night stay)
Fleet Poll (morning) Daily 09:00 Fleet availability snapshot — Blue, Lotus, Lava & Go Car across 1w / 2w / 4w windows; records named sold-out models
Fleet Poll (evening) Daily 21:00 Same as above — twice-daily cadence captures intra-day inventory changes
Fleet Calendar Sweep Every Monday 08:30 12-month availability calendar (Caren + Go Car) + absence inference for Hertz, Avis & Holdur from catalog comparison

The rate scrape, SEO check, and alert check schedule (daily / hourly / weekly) can be changed in Settings. Horizon, seasonal, fleet poll, and fleet calendar jobs always run on their fixed schedules regardless.

What Each View Shows
Rate Intelligence tab — five groups, nine views
NOW
List View
All live rates in a sortable table. Change the date picker to trigger a fresh scrape. The Δ column shows price movement since the previous scrape.
Car Model Matrix
Side-by-side price grid for every canonical model across all competitors. Green = cheapest, red = most expensive. Falls back to the most recent snapshot when the selected dates have no data.
TRENDS
Price History
Time-series line charts showing how prices evolved over the last 7 / 14 / 30 / 90 days. Filter by category, model, or competitor.
Price Changes
Chronological feed of every meaningful competitor price move (default ≥ 5%, configurable). Three layers of intelligence:
• Cumulative Drift bar — net % price movement per competitor over the period (↑ = getting pricier = good for Blue, ↓ = getting cheaper = risk).
• Blue rank badge — each event shows Blue's current market rank for that model (e.g. "#2/6 cheapest").
Select a specific model to switch to a full per-competitor price-history chart.
SEASONAL
Seasonal Analysis
Per-day pricing across the next 12 months (pickup on the 15th, 7-night stay). Three modes: line chart; History (how one month's price evolved across scrapes — clearest yield-management signal); and Gap Map — a heatmap showing Blue's price vs each individual competitor by model × month. Green = Blue charges more than that rival, red = Blue undercuts that rival. Switch granularity between By Category (5×12 grid) and By Model (per-model, per-competitor rows).
FORWARD
Horizon
Per-day pricing for the next 3 / 6 / 12 months, one point per weekly scrape window. Filter by category or pick a specific model for per-competitor lines.
Booking Window
Pick any future pickup date and see how each competitor's price has shifted across successive weekly scrapes as that date approaches. Reveals early-bird vs last-minute pricing strategy. Needs multiple horizon scrapes for the same pickup date to show trends.
COMPETITIVE
Win/Loss Scorecard
Head-to-head comparison of Blue's current prices vs every competitor, per canonical model (threshold ±5% by default). Four layers:
• Best Value Models — top 5 models where Blue has the strongest competitive advantage.
• Needs Attention — models where Blue is undercut by 2 or more competitors simultaneously.
• Win Rate by Category — heatmap grid (competitor × category); click any cell to drill into individual model comparisons.
• Price Scatter — Blue price vs competitor market average per model; dots below the diagonal = Blue is cheaper.
Fleet Pressure
Three layers of fleet intelligence covering all 8 competitors:
• Availability Snapshot — Blue, Lotus, Lava & Go Car Rental polled for real-time class availability across 1w / 2w / 4w forward windows. A red badge shows how many models are sold out; click Poll Now for an immediate refresh (auto-runs 09:00 + 21:00).
• Sold-Out Models panel — named list of every sold-out model per competitor and window, shown as red pills. Filter by window (1w / 2w / 4w).
• 12-Month Calendar heatmap — competitor × month grid colour-coded by availability % (green = open, red = squeezed). Click any cell to see the sold-out model list for that month. Click Sweep 12 Months to refresh (auto-runs every Monday 08:30).
• Absence Alerts panel — inferred sold-out signals for Hertz, Avis & Holdur. These scrapers only return available cars; if a model from the known catalog is missing for a given month, it's flagged as absent. Labelled as "weaker / inferred signal" — use alongside rate data for confirmation.
Settings Reference
What each section in the Settings tab does
API Configuration
SerpAPI Key — required for live Google rank tracking (serpapi.com). Auto-Scrape Schedule — controls rate scrape, SEO check, and alert check cadence (hourly / daily / weekly). Click Save Settings after any change.
Tracked Locations
Add or remove pickup locations. Keflavik Airport and Reykjavik are pre-configured. Note: Lava and Lotus only operate from Keflavik Airport.
Price Alerts
Paste your Slack incoming webhook URL and set the Alert Threshold %. Check Now runs the alert check immediately; Test Webhook sends a test message.
Tracked Competitors
Live/mock status for each of the 8 scrapers with the timestamp of the last successful scrape. Green = live data; gray = mock only.
Category Audit
Lists every car model with its stored vs canonical category. Filter to "Conflicts Only" to find misclassified models and fix via Car Model Mappings.
Car Model Mappings
Maps a competitor's raw model name to a canonical name so it appears correctly in the matrix and charts. Most mappings are automatic — only add here if a model shows up wrong.
Scrape History
Log of every scrape run — manual, scheduled, seasonal, and horizon — showing timestamp, trigger type, location, rates collected, duration, and error status. Click the ⚠ N errors badge on any row to expand and see the full error messages, useful for diagnosing why a specific competitor returned no data.
Tips & Gotchas
  • Mock data badge — if you see "Mock Data" in the top bar, no real scraped data exists for the current filters. Run a scrape first.
  • Fleet Pressure — 4 direct + 3 inferred — Blue, Lotus, Lava, and Go Car Rental return real availability data (Caren API / GoRentals POST API). Holdur, Avis, and Hertz only expose available cars in their HTML responses — their fleet intelligence is inferred by comparing scrape results against a known model catalog built from historical rate data. Absence signals are weaker but still actionable, especially when a model disappears across multiple anchor months. Go Iceland is not polled for fleet pressure (their Cloudflare Worker API does not expose availability flags).
  • Win/Loss needs a recent scrape — the scorecard uses the latest rate snapshot. If your last scrape was more than a day ago, the data may be stale. The availability % badge on each competitor card shows the scrape timestamp.
  • Price Changes & Booking Window need history — these require multiple weekly scrape runs. The more horizon snapshots exist for a pickup date, the richer the lead-time curve.
  • Unknown car models — if a model appears as "Unknown" in the matrix, go to Settings → Category Audit to find it, then add a mapping in Settings → Car Model Mappings.
  • Scrape errors — click the red error badge in Settings → Scrape History to see the full error message for any failed competitor. Common causes: rate limiting (try again later), or the competitor changed their website layout.
  • Insurance tab — click any price cell to edit it inline. Prices persist across restarts. Use Trigger Research to open all competitor insurance pages at once, then Mark Reviewed when done.
  • Dark mode — toggle via the moon/sun icon in the top bar. Preference is saved in your browser.
